### h2: OpenClawの基本操作｜チャットでの指示の出し方
*→ 競合がほぼカバーしていない領域。差別化の核*

#### h3: OpenClawへの指示の基本ルール
- 日本語でOK（LLMが対応しているため）
- 「何を」「どうしてほしいか」を具体的に伝える
- 曖昧な指示より、条件を明確にした方が精度が上がる

具体例の表:

| 指示の出し方 | 精度 | 改善例 |
|------------|------|--------|
| 「メールを整理して」 | △ | 「未読メールを重要度順に並べて、返信が必要なものだけリストアップして」 |
| 「天気教えて」 | ○ | そのままでOK（シンプルなタスクは短い指示で十分） |
| 「レポート作って」 | △ | 「先週のSlackの#salesチャンネルの会話を要約して、箇条書きで5つにまとめて」 |

#### h3: OpenClawの基本コマンド一覧
- `/new` — 新しい会話セッションを開始
- `/skills` — 利用可能なスキル一覧を表示
- `/status` — 現在の状態を確認
- `/help` — ヘルプを表示

#### h3: ブートストラップ（初回の人格設定）のコツ
- 初回接続時に「Hello」と送ると、OpenClawがインタビュー形式で設定を進める
- **コツ**: 短く済ませず、具体的に伝える
  - 名前、職業、主な用途、好みの言語（日本語）、対応トーン
  - 例: 「名前はしゅーへい。Webマーケティング会社で働いています。主にSEO記事の執筆支援と、メール・スケジュール管理の自動化に使いたい。日本語で対応してほしい」
- ブートストラップ後にIDENTITY.md、USER.md、MEMORY.mdが自動作成される
- 🔧 スクショ: ブートストラップのやりとり画面

### h2: OpenClawの主要機能と使い方
*→ 機能別に「概要 → 設定方法 → 使用例」の3段構成*

#### h3: OpenClawのメモリ機能｜使うほど賢くなるAI
- **概要**: ユーザーの名前・好み・仕事のやり方を記憶し、使うほど最適化される
- **仕組み**: MEMORY.md、USER.md等のファイルに情報が蓄積
- **設定方法**: ブートストラップ時に自動設定。追加情報はチャットで「覚えておいて: ○○」と伝える
- **活用例**: 「レポートは箇条書き5項目で」と1度伝えれば、以降は自動で適用

#### h3: OpenClawのHeartbeat機能｜定期チェックの自動化
- **概要**: 設定した間隔（デフォルト30分）でOpenClawが自動的にチェックを実行
- **できること**: メール監視、カレンダー確認、Webページの変更検知
- **設定方法**: ワークスペースの`HEARTBEAT.md`ファイルに指示を記述
  - 例: 「30分ごとに未読メールをチェックし、緊急のものがあればTelegramに通知して」
- **Heartbeatとcronの違い**: 表で比較

| 項目 | Heartbeat | cronジョブ |
|------|-----------|----------|
| 用途 | 継続的な監視 | 特定時刻の実行 |
| 間隔 | デフォルト30分 | cron式で自由設定 |
| 設定方法 | HEARTBEAT.mdファイル | CLIまたはチャットで作成 |
| 適した例 | メール監視、カレンダー確認 | 朝のブリーフィング、週次レポート |

[出典: 公式ドキュメント https://docs.openclaw.ai/automation/cron-vs-heartbeat]

#### h3: OpenClawのcronジョブ｜定時タスクの自動実行
- **概要**: 指定した時刻にタスクを自動実行する機能
- **設定方法**: チャットで「毎朝7時にニュース要約をSlackに送って」と指示するだけでOK
  - または `openclaw cron create` コマンドで作成
- **cron式の書き方**: 基本パターンの表

| やりたいこと | cron式 | 意味 |
|------------|--------|------|
| 毎朝7時 | `0 7 * * *` | 毎日07:00に実行 |
| 平日の朝8時 | `0 8 * * 1-5` | 月〜金の08:00に実行 |
| 毎週月曜9時 | `0 9 * * 1` | 毎週月曜09:00に実行 |
| 2時間ごと | `0 */2 * * *` | 2時間ごとに実行 |

- **管理コマンド**: `openclaw cron list`（一覧）/ `openclaw cron delete`（削除）

#### h3: OpenClawのSkills（スキル）｜機能を拡張する方法
- **概要**: AIに特定タスクの実行方法を教える「教科書」のようなもの
- **3種類のスキル**: バンドル（53種類が最初から入っている）/ ClawHub（13,700+の追加スキル）/ 自作
- **おすすめの初期スキル**:

| スキル名 | 用途 | おすすめ度 |
|---------|------|----------|
| gog | Google Workspace統合（Gmail/Calendar/Drive） | ★★★ |
| weather | 天気情報の取得 | ★★★ |
| summarize | テキスト・ドキュメントの要約 | ★★★ |
| github | GitHub操作の自動化 | ★★☆（開発者向け） |
| blogwatcher | RSSフィードの監視 | ★★☆ |

- **ClawHubからのインストール方法**: チャットで「ClawHubからweatherスキルをインストールして」と指示
  - または `clawhub install weather` コマンド
- **セキュリティ注意**: ClawHubのスキルは約7%に脆弱性が報告されている。インストール前に `clawhub inspect <slug>` で確認を推奨
- **→ スキルの詳細は「OpenClaw Skills」記事で解説** + skills記事への内部リンク

#### h3: OpenClawのマルチエージェント｜用途別にAIを使い分ける
- **概要**: 1つのOpenClawインスタンスで複数のエージェントを運用し、チャンネルやアカウントごとに使い分ける
- **活用例**: Slack#devチャンネル → 開発用エージェント / Slack#marketingチャンネル → マーケティング用エージェント
- **設定方法**: openclaw.jsonの`agents`セクションで複数エージェントを定義
- **難易度**: ★★★（中級者向け。まずは1エージェントで慣れてから）

### h2: 目的別OpenClawの使い方レシピ
*→ 「やりたいこと」から逆引きできる実践セクション。差別化の核その2*

#### h3: レシピ①｜毎朝のブリーフィングを自動配信する
**完成イメージ**: 毎朝7時に、天気・今日の予定・未読メール要約がTelegramに届く

**設定手順**:
1. Google Workspaceスキル（gog）をインストール → Google連携を設定
2. weatherスキルをインストール
3. チャットで以下を指示:
   「毎朝7時に、今日の天気予報、Googleカレンダーの今日の予定、未読メールの要約をまとめてTelegramに送って」
4. OpenClawがcronジョブを自動作成

**ポイント**: 最初は天気だけなど小さく始めて、徐々に項目を追加するのがおすすめ

🔧 スクショ: 実際に届いたブリーフィングメッセージの画面

#### h3: レシピ②｜未読メールを自動で分類・要約する
**完成イメージ**: 30分ごとにメールをチェックし、重要なものだけ通知してくれる

**設定手順**:
1. Google Workspaceスキル（gog）が設定済みであることを確認
2. HEARTBEAT.mdに以下を記述:
   「30分ごとに未読メールをチェックして、以下のルールで分類してください:
   - 緊急（24時間以内の対応が必要）→ すぐにTelegramに通知
   - 重要（今週中に対応）→ 1日1回まとめて通知
   - 情報（読むだけ）→ 週次サマリーに含める
   - 不要（メルマガ等）→ アーカイブ」

**ポイント**: 分類ルールは自分の業務に合わせてカスタマイズする

#### h3: レシピ③｜Webサイトの更新を自動監視する
**完成イメージ**: 指定したWebページに変更があったら通知してくれる

**設定手順**:
1. チャットで以下を指示:
   「以下のURLを毎日チェックして、内容に変更があればTelegramに知らせて: [URL]」
2. OpenClawがcronジョブまたはHeartbeatで自動監視を開始

**活用シーン**: 競合サイトの価格変動、公式ドキュメントの更新、採用ページの変更検知など

#### h3: レシピ④｜議事録からタスクを自動抽出する
**完成イメージ**: 会議のテキストを送ると、アクションアイテムを抽出してリスト化

**設定手順**:
1. 会議後にOpenClawに議事録テキストを送信
2. 「この議事録からアクションアイテムを抽出して、担当者・期限付きのリストにまとめて」と指示
3. Google CalendarやTrelloと連携していれば、そのままタスク登録も可能

### h2: OpenClawを使いこなすための5つのコツ

#### h3: ①小さく始めて徐々に拡張する
- 最初から全機能を使おうとしない
- まずは1つのユースケース（例: 天気通知）で感覚を掴む
- 慣れたら機能やスキルを追加していく

#### h3: ②指示は「具体的に」「条件付きで」出す
- 「いい感じにして」ではなく「箇条書き5項目で、日本語で」
- 条件を明確にするほど、期待通りの結果が返ってくる

#### h3: ③セキュリティ設定を忘れずに
- サンドボックスモードの有効化
- 最小権限の原則（必要な権限だけを付与）
- APIキーの安全な管理（.envファイルは外部に公開しない）
- **→ セキュリティの詳細は「OpenClaw セキュリティ」記事で解説** + セキュリティ記事への内部リンク

#### h3: ④コストを意識してモデルを使い分ける
- 簡単なタスク → 低コストモデル（Haiku等）
- 複雑な分析・長文生成 → 高性能モデル（Sonnet、Opus等）
- モデルルーティングで自動切り替えも可能
- 支出ガードレール設定で月額上限を設定しておく

#### h3: ⑤うまくいかない時は「/new」でリセット
- 会話が混乱したら`/new`で新しいセッションを開始
- エラーが続く場合はログを確認（`openclaw logs`）
- 最新バージョンへのアップデートで解決することも多い

### h2: OpenClawの使い方でよくあるトラブルと対処法

表形式で整理:

| トラブル | 原因 | 対処法 |
|---------|------|--------|
| 指示しても反応がない | ゲートウェイが停止している | `openclaw status`で確認。停止していれば`openclaw gateway start` |
| 日本語が文字化けする | LLMモデルの設定ミス | Claude or GPTが設定されているか確認 |
| cronジョブが動かない | タイムゾーンの設定ミス | VPSのタイムゾーンがUTCの場合、日本時間に変換して設定 |
| API料金が想定以上 | 高コストモデルの常時使用 | モデルルーティングで低コストモデルを活用 + 支出ガードレール設定 |
| スキルが動作しない | 前提条件が未設定 | スキルの3条件（Configuration / Installation / Authorization）を確認 |
| 「LLM request rejected」エラー | セッションの問題 | `/new`でリセット |
