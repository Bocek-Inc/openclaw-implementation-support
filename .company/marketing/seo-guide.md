# SEO記事の書き方ガイド（社内ナレッジ）

## 基本方針

- 「SEO的に評価される記事構成を守ること」と「質の高いコンテンツを作成すること」の**両立**を目指す
- 質の高いコンテンツ＝執筆KWの持つ「ユーザーニーズ」を充足できるコンテンツ
- OpenClawのKWは「個人ユーザー」と「企業ユーザー」が混在 → それぞれのニーズを考察し、網羅的な記事を作成

---

## SEO的に評価される記事構成

### 見出しの使い方

- h2 > h3 > h4 の階層構造を守る
- 例）
  - h2 早起きするメリット3選
    - h3 頭がさえる
    - h3 生活リズムが整う
    - h3 朝はモーニングメニューが食べれる

### 構成の決め方

#### 観点① 網羅性

- 競合メディアが用意している見出し（情報）は**全て担保**する
- ただし、狙いたい層（ペルソナ）から大きくずれる情報は間引いてOK
- **同じだけの網羅性を担保してからが勝負**

#### 観点② オリジナルコンテンツ

- 競合が勝てない記事にするための差別化ポイント
- 検索volや対策難度が高いKWでは、DR（ドメインレート）が高い or 圧倒的にクオリティが高いコンテンツが必要
- オリジナルコンテンツ＝他メディアが対策していないが、ニーズがある情報
- **例：**
  - 「使っている風景」のスクショ
  - 「使ってみた所感」
  - 有識者としての自分なりの見解（記事を書くうちに知見が溜まる）
- ネタ探しのソース：知恵袋、Twitter（X）、ChatGPT、自分の頭

## 記事の書き方

### ステップ① 導入文

1. **easyな共感** = ユーザーが自分でわかっている表層の気持ち
   - 例）「メール作成をChatGPTに代行させるにはどうすればいいんだろう？」って気になりますよね。

2. **coreな共感** = そのKWを検索するに至った根本課題
   - 例）「メールの作成で貴重な朝の時間や日常業務の時間が減ってしまうのしんどいですよね…」

3. **読者を引き込む言葉**
   - 例）「今回紹介するプロンプトとその書き方を習得すれば、その時間を毎日たった15分に短縮できます」

4. **記事の概要**
   - 例）今回は、ChatGPTでメールを作成するためのプロンプトと使い方をご紹介します。

※ 最近のSEO記事では必ずしもこのフォーマットに従っていない場合も。柔軟に工夫が必要。

### ステップ② 記事構成

#### 分かりやすさ
- 極論、見出しを見ただけで「何が書いてあるか大体予想できる」レベルにする

#### 強調の優先度
- **太文字+黄色マーカー** ＞ **太文字** ＞ 普通の文字
- 黄色マーカーのカラーコード：`#FEE8AB` で統一

#### 文章量・読みやすさ
- 文章を3行ごとなどのまとまりで区切り、スペースを持たせる
- **一見出しあたり300字が限度。超えると重い**
- 「ここ内容切れるな～」ってところはしっかり改行
- 冗長な表現を避ける。同じ内容なら文字数が少ない方を選ぶ

#### 文体・トーン
- やさしく丁寧に語りつつ、親しみのある雰囲気
- 基本は「です」「ます」調
- **NG表現：**
  - 「〜しちゃおう！」「〜べき！」
  - 話し言葉（めっちゃ、とか）
  - 動詞に「〜いく」「〜くる」
    - NG「説明していこうと思います」→ OK「説明しようと思います」
    - NG「推移していくのか」→ OK「推移するのか」
  - 過度な誇張表現（画期的、革新的、絶対〜する等）
  - 感情を煽る言葉

#### 表記ルール
- **英字・数字・スペースは半角**
- **記号は全角**：（）！？「」
- 固有名詞は例外（例：「戦国七雄」）
- メディアの権威性を保つための統一

#### 文章テクニック
- **結論ファースト**（少し英語っぽい語り方）
  - NG「Deep Researchは、OpenAIが2025年2月に発表したChatGPT向けの新機能で、従来のChatGPTのように即座に回答する対話型AIを超えて、ウェブ検索や情報収集、分析までを一括で実行し…」
  - OK「Deep Researchは、OpenAIが2025年2月に発表したChatGPT向けの新機能で、オンライン調査やデータ解析を自動で行うAIエージェントです。」
- **修飾語は直前に置く**
  - NG「急激に一酸化炭素がヘモグロビンと結びつき…」
  - OK「一酸化炭素がヘモグロビンと急激に結びつき…」
- 削れる言葉は削る（「そのように考えてみると」→「そう考えると」）

#### 番号振りルール
- 手順・項目数が3〜4個を超える場合は番号をつける
- 表記は「①」「②」「③」…で統一
- 見出しに「〜つの」を入れる（例：「早起きをする3つのメリット」）
  - ユーザーが○○個あることを意識して読める
  - 分かりづらくなる場合は例外

### ステップ③ 引用・参考・内部リンク

#### 内部リンク
- 重複内容は記事リンクで回遊してもらう
- **階層を意識**：上階層・下階層の記事は原則必ず貼り付ける
- ※OpenClawは新興KWが多いので、今後記事が増えたときに特に意識

#### 外部リンク（重要）
- **競合メディア（not一次情報）への発リンクはNG**
  - 理由：情報源→競合→自社 の流れになり、記事作成の価値がなくなる＋競合に評価を渡す
- **一次情報への発リンクは推奨**
  - knowクエリ（〜とは）に関する発リンクはプラス
  - 権威性のあるサイトの引用は効果的
  - 推奨引用先：
    - Xでのバズった投稿（Trump、Sam Altmanなど）
    - 公式サイト（OpenAI、Microsoftなど）
    - 政府・公的機関

### ステップ④ まとめ

- **まとめは必須**
- 必ず入れるもの：
  - 記事内容の振り返り
  - 「今回〜〜を紹介してきましたが、いかがだったでしょうか。」

## 注意点チェックリスト

- [ ] 誤字脱字はないか（**SEO的にかなりアウト**）
- [ ] 表記ゆれはないか（株式会社orブランド名、英語or日本語、カタカナorひらがな）
- [ ] 環境依存文字は使用していないか
- [ ] 数字/英語は半角表記になっているか
- [ ] 不要なスペース/改行は入っていないか
- [ ] 誇張表現、最上級/最大級表現、断定表現、差別表現、宗教/風水への言及はないか
- [ ] h2/h3/h4の順番ルールが守られているか
- [ ] 画像は説明の補助として適切に使われているか
  - 手順・やり方の説明にはスクショ推奨
  - 情報が多い場合は表の方が良い（画像はクロール対象外、文字は読まれる）

## コンテンツSEO実践ガイド（才流ナレッジ）

> 出典: https://sairu.co.jp/method/93847/（2026-03-17 追記）

### コンテンツSEOの目的

1. **見込み顧客との継続的関係構築** - 検索ユーザーに価値ある情報を提供し、ブランド認知とエンゲージメントを構築
2. **広告費に依存しない安定的集客** - 一度上位表示すれば継続的に流入を獲得でき、費用対効果が高い

### SEO施策の優先順位

**コンテンツSEO → 外部SEO → テクニカルSEO** の順で実施する。

### キーワードプランニング

#### キーワード洗い出し方法

| 方法 | 説明 |
|------|------|
| リスティング広告の成果KW確認 | 既にCVしているKWを優先 |
| 競合サイトの流入KW分析 | Ahrefs、SE Ranking等で調査 |
| GA4でCV獲得記事を特定 | 実績ベースで有望KWを発見 |
| 関連KWの派生展開 | 「○○ 評判」→「○○ 口コミ」「○○ メリット デメリット」等 |
| 生成AI活用 | ChatGPT等に商材・ペルソナ・CV目的を伝えてKW案出し（必ず検証） |

#### キーワード抽出目安

- **1年で36〜60ワード**、可能なら3年分を用意する

#### キーワードのグルーピング分類

| 分類 | 内容 | 検索意図の段階 |
|------|------|--------------|
| A. 基礎理解 | 「○○とは」系 | 認知段階 |
| B. 比較検討 | 「○○ 比較」「○○ おすすめ」 | 比較〜検討段階 |
| C. 商標深掘り | 特定製品の詳細情報 | 検討段階 |
| D. 導入・運用 | 「○○ 使い方」「○○ 設定」 | 導入段階 |
| E. 成功事例 | 「○○ 導入事例」「○○ 活用事例」 | 検討〜導入段階 |
| F. 関連戦略 | 周辺テーマの戦略記事 | 認知〜比較段階 |
| G. リスク・懸念 | 「○○ デメリット」「○○ 注意点」 | 比較段階 |
| H. 行動促進 | 「○○ 導入手順」「○○ 無料トライアル」 | 導入段階 |

### 競合調査

#### 確認すべき項目

| 項目 | 確認ポイント |
|------|------------|
| Meta title | KWの配置位置と使い方 |
| 記事構成（h1〜h3） | 見出しの網羅性・構造 |
| 文字数 | 上位3記事の平均が目安 |
| 画像・動画 | 使用状況、種類、配置 |
| 関連KW | カバーしているサブトピック |
| 検索ナビゲーション | 画像・動画・ニュース等の表示順序 |
| リッチスニペット | 画像・ショッピング主流の場合テキストでの上位は困難 |

#### 見なくてよい項目

- **Meta keywords** - 2009年以降、評価に使われていない
- **Meta description** - 順位に影響なし（ただしCTRには影響）
- **OGP** - 順位に影響なし

### コンテンツ制作ペース

- **初期目標**: 3ヶ月で約10記事
- **継続ペース**: 月3〜5記事

### 効果測定と改善

#### 計測ツール

| ツール | 計測対象 |
|--------|---------|
| GA4 | CVR測定 |
| Google Search Console | 検索順位、CTR |

#### 改善施策の対応表

| 改善対象 | 施策 | 具体的内容 |
|---------|------|----------|
| 検索順位 | 記事リライト | 内容充実、検索意図への対応 |
| クリック率 | Meta title改善 | 魅力的で具体的なタイトルに |
| CV率 | 導線改善 | CTA配置、フォーム簡略化 |

### リライト戦略

#### リライト手順

1. 最新上位3位のコンテンツを確認
2. 自社記事との情報差分を把握
3. 上位サイトにない情報を追加執筆
4. 自社独自情報・事例を付与
5. 可読性を図解で改善
6. 公開
7. **2週間後に計測**

#### リライトすべき3つのタイミング

| タイミング | 状況 | 対応方針 |
|-----------|------|---------|
| ① 順位安定後、目標未達 | 公開2週間後に安定したが4位以下 | 上位記事より詳しく、構成を見直し |
| ② 急激な順位低下 | 1週間様子見後も回復なし | 検索意図の変化確認、最新情報更新、構成再構築 |
| ③ 定期チェック | 半年〜1年ごと | データ鮮度・トレンド反映度・競合比較を確認 |

#### 効果確認サイクル

計測 → 改善点特定 → 施策実施 → 2週間後に再計測 → 結果分析 → 再施策

### KPI指標

- キーワード検索順位
- 月間オーガニック流入数
- コンバージョン率/実数
- ページ滞在時間
- 月間コンテンツリリース数
- 検索結果CTR

### よくある失敗と対策

| 失敗パターン | 対策 |
|-------------|------|
| 短期で諦める | 開始〜1年は成果が出にくい。ターゲット再検討、検索意図深掘り、コンテンツ再設計で粘る |
| 記事管理をしない | 6ヶ月〜3年で重複記事が発生。KW・記事対応表を日頃から管理。重複時は301リダイレクトで統合 |
| 情報品質を軽視する | 低コスト大量生産は不可。中長期視点でユーザーに向き合う |

### 生成AI利用の方針（2025年6月時点の業界見解）

- **AI 100%執筆での上位表示狙いは非推奨**
- ファクトチェックは必須
- Googleの基本方針：「制作手法問わず有用なコンテンツを評価」
- **推奨される活用方法**: KW出し、データ整形、文章校正、矛盾点指摘、批判可能性の列挙
