# 動画素材1: OpenClaw 究極の初心者ガイド

## 基本情報
- **URL**: https://www.youtube.com/watch?v=st534T7-mdE&t=627s
- **内容**: OpenClawの概要から実際のセットアップ、運用までを網羅した超入門ガイド

## 要約（17セクション）

### 1. OpenClawとは何か
- 「あなたのところに来るAIアシスタント」（ChatGPT等は"訪ねる"ツール）
- サーバー上で24/365稼働し、自律的に行動
- 3つの柱：①頭脳と記憶（LLM+永続メモリ） ②常時稼働 ③ツール/アクション連携
- 用途例：毎朝のタスクまとめ送信、面接準備資料の自動作成、ジム予約の自動化
- 名前の変遷：Clawdbot / Moltbot → OpenClaw

### 2. 動作環境の選択肢（3つ）
- **自分のPC**: 無料だがPC OFF で停止、セキュリティリスクあり
- **予備ハードウェア（Mac miniなど）**: 常時稼働・隔離可能だが初期投資500ドル〜
- **VPS/サーバー（推奨）**: 月数ドル〜、24/7、問題がサーバー側で完結

### 3. Hostingerでのデプロイ
- OpenClaw用テンプレートでワンクリックデプロイ
- プラン目安：KVM1（小規模）、KVM2（推奨）、KVM4（ローカルモデル向け）
- 自動バックアップ推奨（月3ドル程度）

### 4. セットアップ後の重要ポイント
- ゲートウェイトークン＝マスターキー → パスワードマネージャーに保存
- LLM APIキー設定（Anthropic Claude推奨：40ドルチャージでTier 2）
- 支出ガードレール設定（月額上限、自動リロードOFF）

### 5. 初回ブートストラップ
- Bootstrap.mdが起動し、ボットの性格・役割・ユーザー情報を設定
- 短く済ませず、具体的に説明するのがコツ
- ChatGPT/Claudeとの違い：OpenClawは永続メモリで使うほど賢くなる

### 6. セキュリティ強化
- OpenClawはターミナルコマンド・ファイル読み書き・Web閲覧が可能 → リスクあり
- セキュリティドキュメントURLをボットに渡して自己設定させる
- 最小権限の原則：メッセージ送信・ファイル削除・ネットワーク要求は事前承認制に
- ガードレール：タスク3回失敗で停止、実行時間上限10分
- 小さく始める（Telegramだけ、スキル1〜2個から）

### 7. Telegram連携
- BotFatherでボット作成 → APIトークン取得 → OpenClawに接続
- ペアリングコードで自分のユーザーIDを許可リストに登録
- 15+プラットフォーム対応（WhatsApp/Discord/Slack等）

### 8. スキル（機能拡張プラグイン）
- ClawHub（公式マーケット）またはTelegramから入手
- セキュリティ注意：300以上の悪意あるスキル発見報告あり
- VirusTotalレポート確認、ボットにコードレビューさせることを推奨

### 9. Google Workspace連携（OAuth設定）
- Google Cloud Consoleでプロジェクト作成 → API有効化 → OAuth設定
- 所要10〜15分、一度やれば済む
- Gmail/カレンダー/Drive/Sheets/Docs/People APIに対応

### 10. ワークスペース構造
- 主要ファイル3つ：
  - Agents.md（行動ルール）
  - Soul.md（性格・人格）
  - User.md（ユーザー情報）
- Memory.md（永続メモリ）
- 圧縮メモリフラッシュ＆セッションメモリの有効化推奨

### 11. 自動化（cronジョブ/ハートビート）
- cronジョブ：決まった時間に実行（毎朝のブリーフィング等）
- ハートビート：30分ごとの定期チェック（緊急メール監視等）
- ハートビートはトークン消費大 → 安価モデルで実行すべき

### 12. モデルとコスト設計
- Tier 1（Opus等）: 複雑な推論、1プロンプト2〜6ドル
- Tier 2（Sonnet等）: 日常タスク
- Tier 3（Haiku等）: ルーチン、Opusの1/25コスト
- 無料モデル（NVIDIA等）: 最終フォールバック用
- 3大コスト落とし穴：全処理Tier 1、再試行ループ、高価モデルでのハートビート

### 13. 音声入出力
- STT: OpenAI Whisper利用
- TTS: Edge TTS（無料、74言語、300+音声）

### 14. サブエージェント
- 並列調査（例：3プラットフォーム同時比較）
- 段階的パイプライン（調査→分析→レポート）
- Brave Search API推奨（月5ドル無料枠あり）

### 15. アップデートとトラブル復旧
- 週に複数回アップデートあり
- 復旧手段：ボットに停止指示 → Docker停止 → APIキー取り消し（最終手段）
- 自動バックアップ/スナップショットからの復元

### 16-17. まとめ・次にやること
- 小さなタスクから始めて関係性を育てる
- User.md/Soul.mdを少しずつ深める
- 慣れたら追加スキル・高度な自動化へ拡張
