# OpenClaw: Heartbeat vs Cron の使い分け

## 概要

OpenClawには定期実行の仕組みが2つある。用途によって使い分けることで、トークン消費を最小化しつつ確実にタスクをこなせる。

## Heartbeat（HEARTBEAT.md）

### 向いているケース
- 複数チェックをまとめてバッチ処理したい（メール確認 + カレンダー + Slack通知を1ターンで）
- 直近のセッション文脈が必要なタスク（会話の流れを把握した上でのアクション）
- タイミングが多少ずれてもいいもの（30分に1回程度）
- 終わったら止める一時的な監視タスク（例：IssueがCloseされたら設定を戻す）

### 使い方
`HEARTBEAT.md` に監視タスクを書くだけ。エージェントがハートビートポーリング時に読み込んで実行する。

```markdown
## 定期チェック
- [ ] openclaw/openclaw#XXXX がクローズされたら設定を戻す
```

### 注意点
- 「メンタルメモ」はNG、**必ずファイルに書く**（セッション跨ぎで消えるため）
- HEARTBEAT.md が空 or コメントのみ → `HEARTBEAT_OK` を返してスキップ

---

## Cron（openclaw cron）

### 向いているケース
- 正確なタイミングが必要（「毎朝9時ぴったり」）
- メインセッションから独立して動かしたいタスク
- 異なるモデル・thinking レベルで実行したいタスク
- 単発リマインダー（「20分後に知らせて」）
- 出力をチャンネルに直接届けたいタスク（日報投稿など）

### 設定例
```bash
openclaw cron add \
  --schedule "0 8 * * *" \
  --task "日報を作成して #channel に投稿" \
  --model anthropic/claude-sonnet-4-20250514
```

### 主なオプション
| オプション | 説明 |
|-----------|------|
| `--schedule` | cron式（分 時 日 月 曜日） |
| `--task` | 実行する指示文 |
| `--model` | 使用モデル（省略でデフォルト） |
| `--channel` | 出力先チャンネル |

---

## 使い分けチートシート

| 条件 | 使うべき仕組み |
|------|-------------|
| 複数チェックをまとめたい | Heartbeat |
| 正確な時刻指定が必要 | Cron |
| 会話文脈が必要 | Heartbeat |
| 単発リマインダー | Cron |
| 終わったら止める監視 | Heartbeat（タスクをコメントアウトするだけ） |
| チャンネルへの直接投稿 | Cron |

---

## 実例：Bocek社内運用

- **日報投稿（毎朝8時）** → Cron
- **Slack使用量集計（23:55）** → Cron（Pythonスクリプト実行）
- **OpenClawバグ修正の監視** → HEARTBEAT.md に記載（修正されたら削除）
- **メール・カレンダー確認** → HEARTBEAT.md にまとめて記載
