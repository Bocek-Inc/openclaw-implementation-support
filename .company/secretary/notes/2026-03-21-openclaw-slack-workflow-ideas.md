# OpenClaw × Slack × Mac mini 社内活用アイデア

## 方針決定

### 記事執筆と社内運用の棲み分け
- **「使い方」記事** → VPSベースで一般的な例を執筆
- **会社の実務運用** → Mac miniで常時稼働
- **「mac mini」記事** → 別記事として進行中

### 環境選定
- 社内実務は全部Mac miniで運用する
- 理由: 常時稼働前提、社内DB/リポへのアクセスはローカルが安全、VPSにアクセス権を持たせるほうがリスク高い

## 社内でやりたいこと一覧

### ◎ 相性抜群（すぐ着手可能）
1. **要求リスト(notion)への追加（重複チェック・内容整理）**
   - Slack対話で要求登録 → 類似要求の重複チェック → 内容整形 → DB追加
   - 現状の課題: 重複発生、内容がわかりにくくなる
2. **QAリストの作成**
   - 仕様から自動生成
3. **デグレ確認QAの更新**
   - 仕様サイト参照 → 既存機能のQA項目を更新
4. **仕様確認 → 仕様サイトに追記**
   - Slack対話で確認 → 自動反映

### ◎ 開発プロセス自動化（追加 2026-03-21）
5. **要件定義の自動作成**
   - 要求リストから要件定義を自動生成
   - 既存の仕様と照らし合わせながら整合性チェック
   - 機能要件・非機能要件を網羅
6. **開発チケットの自動作成**
   - 要件定義から開発チケットを自動生成
   - 手順の詳細まで記載
7. **レビューの一次対応**
   - PRのコードレビューを自動で一次チェック
   - 指摘事項をSlackに報告、人間が最終判断

### ◎ プロジェクト管理（追加 2026-03-21）
8. **日程調整**
   - メンバーの空き状況確認 → 候補日提示 → 確定
9. **工数見積**
   - 要件・チケットから工数を自動見積もり

### ◎ 運用・保守（追加 2026-03-21）
10. **アラートに対して自動対応**
    - Sentry等のアラート検知 → 原因調査 → 修正PR or 報告
11. **依存関係の修復（Dependabotの代わり）**
    - 依存パッケージの更新検知 → 互換性チェック → 更新PR作成
12. **技術選定・リサーチ・ナレッジ追加検索**
    - Slackで「○○の技術比較して」→ 調査 → ナレッジに蓄積

### ◎ ブラウザ操作・QA自動化（追加 2026-03-22）
13. **E2Eテストの自動実行**
    - 自然言語でテストシナリオ定義 → ブラウザ操作で自動実行
    - セルフヒーリング機能付き（UIが変わっても自動適応）
    - Playwright + CDP ベースで高速・正確
14. **デグレ確認の自動実行**
    - 仕様サイト参照 → ブラウザ操作で既存機能を実際に確認
    - エラーがあればスクショ付きでSlackに報告
15. **定期的な死活監視**
    - Cron + ブラウザでWebアプリに定期アクセス → 異常検知 → Slack報告
16. **デバッグ支援**
    - コンソールログ取得、ネットワーク監視、パフォーマンス計測（Core Web Vitals）
    - Slackから「ログイン画面でエラー出てるか確認して」→ スクショ+ログ付きで報告

### ○ 可能（仕組み構築が必要）
17. **自動テスト実行（ユニットテスト等）**
    - コマンド実行 → Slackに結果報告
18. **お客様用仕様サイトの更新**
    - コード変更 → デプロイ
19. **リリース配信**
    - 承認フロー付きにすべき（自動実行は危険）

### 要求リスト管理のイメージ
```
ユーザー: 「○○の機能追加要求を登録して」
OpenClaw: 「類似の要求が2件あります:
           - #123: ○○の△△対応（2026-02-15）
           - #156: ○○の□□改善（2026-03-01）
           統合しますか？新規登録しますか？」
ユーザー: 「#123と統合して」
OpenClaw: 「統合しました。整理した内容: [要求文]。OKですか？」
```

## 技術的な裏付け（リサーチ結果より）

### OpenClaw × Claude Code連携
- Claude Code Skill（MCP経由）が最も整備された方式
- 複数リポジトリは「1エージェント + GitHub MCP」方式が最もシンプル
- バグ報告→自動PR作成が15〜30分で完了する実践事例あり

### セットアップ工数（初心者目安）
- OpenClaw基本セットアップ: 1.5〜2.5時間
- Claude Code Skill設定: 1.5〜2.5時間
- 複数リポジトリBinding: 2.5〜4.5時間
- 合計: 5.5〜9.5時間（1〜2日）

### Mac mini運用
- `openclaw --install-daemon` で自動起動・クラッシュ自動再起動
- Cloudflare Tunnel（無料）でポート開放不要
- 電気代: 月200〜400円程度

## 着手順序（案）
1. Mac miniにOpenClawセットアップ → Slack連携

3. ナレッジを貯めながら記事ネタにする
4. 他のワークフローを順次追加
