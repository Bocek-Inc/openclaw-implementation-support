# OpenClawデモ実行で得た知見（2026-03-15）

## 実施したデモ

### ✅ ブートストラップ（初回セットアップ）
- 「こんにちは」で開始、日本語でやりとり可能
- OpenClawが自分で名前を決めた（「クロウ」）→ キャラクター性がある
- 口調・スタイルの設定がすぐ反映される
- IDENTITY.md、USER.md、MEMORY.md等を自動作成 → 永続メモリの実感

### ✅ ファイル作成
- ワークスペースへのファイル作成は問題なく動作
- **重要な発見**: VPSの場合、ユーザーの手元PCのデスクトップには直接アクセスできない
  - OpenClawから丁寧に説明してくれた（「俺はDockerコンテナの中で動いてるから」）
  - 代替案も提示してくれた（ワークスペースに作る or ノード連携）
  - → 記事にVPS vs Mac miniの違いとして追記済み

### ✅ Telegram連携
- BotFather → ボット作成 → トークン取得 → OpenClawに設定 → ペアリング → 接続完了
- 手順自体はスムーズ
- 再起動後に「LLM request rejected: thinking blocks」エラーが出た → `/new`でリセットして解決

### ❌ cronジョブ（断念）
- **問題1**: webchatだとdelivery.channelの指定が必要 → OpenClawが自分でエラーを検知して修正してくれた
- **問題2**: Hostingerのタイムゾーン設定がUTCのまま → 日本時間の指定が正しく動かない可能性
  - Hostingerでサーバーロケーション「Tokyo」が選択可能かは未確認（購入後は変更不可）
- → デモとしてはユーザーにすぐ結果を見せられないため断念
