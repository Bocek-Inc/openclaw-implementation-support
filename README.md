# OpenClaw 導入支援事業 - コンテンツ管理リポジトリ

OpenClaw導入支援事業の立ち上げフェーズにおける、リサーチ・記事制作・プロジェクト管理を一元化するリポジトリです。

## このリポジトリについて

このリポジトリは、OpenClaw（オープンソースのAIコーディングエージェント）の導入支援事業を推進するためのコンテンツ管理基盤です。

Claude Code上で動作する「仮想カンパニー」というフレームワークを使い、秘書・CEO・各部署のロールを持ったAIアシスタントが、記事制作やリサーチを組織的にサポートします。

### 主な用途

- **SEO記事の企画・執筆・管理**: OpenClawに関するSEO記事を体系的に制作
- **リサーチ・ナレッジ管理**: 市場調査、競合分析、技術調査の蓄積と整理
- **プロジェクト管理**: 記事制作プロジェクトの進捗・タスク管理
- **レビュー・改善サイクル**: 記事品質の継続的な改善

## ディレクトリ構成

```
.
├── .company/                    # 仮想カンパニー（メインの作業ディレクトリ）
│   ├── CLAUDE.md                # 組織全体のルール・設定
│   ├── secretary/               # 秘書室: 窓口・TODO管理・壁打ち
│   ├── ceo/                     # CEO: 意思決定・部署振り分け
│   ├── pm/                      # PM: プロジェクト進捗管理
│   ├── research/                # リサーチ: 市場調査・技術調査・ナレッジ蓄積
│   │   ├── topics/              #   調査トピック（情報源ベースの一次記録）
│   │   └── knowledge/           #   テーマ別統合ナレッジ
│   ├── marketing/               # マーケティング: SEO記事制作 ★最優先
│   │   ├── content-plan/        #   記事の企画書
│   │   ├── drafts/              #   下書き〜清書
│   │   ├── published/           #   公開済み記事（HTML）
│   │   └── revisions/           #   リライト管理
│   ├── sales/                   # 営業: クライアント管理（将来用）
│   └── reviews/                 # レビュー: 週次・月次振り返り
├── .claude/                     # Claude Codeの設定・カスタムスキル
│   └── skills/                  # /article, /proposal などのスキル定義
├── docs/                        # 初期リサーチ報告書
└── README.md
```

## 仮想カンパニーの仕組み

このリポジトリでは、Claude Codeの `/company` コマンドを使って仮想的な「会社組織」を運用しています。

```
━━━━━━━━━━━━━━━━━━━━
  オーナー（あなた）
━━━━━━━━━━━━━━━━━━━━
         │
    ┌────┴────┐
    │  CEO    │  ← 振り分け・意思決定
    └────┬────┘
         │
  ┌──────┼──────┬──────┐
  │      │      │      │
秘書室   PM   リサーチ  │
（常設）         ┌──┴──┐
              マーケ  営業
           （★最優先）
```

### 使い方

1. このリポジトリのディレクトリでClaude Codeを起動
2. `/company` と入力すると秘書が応答
3. やりたいことを伝えると、秘書が適切な部署に振り分けて対応

```
# 例: 記事を書きたい場合
/company
> 「OpenClawのセキュリティについて記事を書きたい」

# 秘書 → CEO振り分け → リサーチ部門がナレッジ準備 → マーケ部門が執筆
```

### カスタムスキル

| コマンド | 説明 |
|---------|------|
| `/company` | 秘書に相談（何でもOK） |
| `/article` | SEO記事の執筆ワークフロー（リサーチ→構成→執筆→レビュー→公開準備） |
| `/proposal` | 会話コンテキストから上司・チーム向けの提案文を自動生成 |

## 記事制作のワークフロー

SEO記事は以下のフローで制作しています。

1. **KW選定・リサーチ**: `marketing/kw-list.md` からKWを選定 → Web検索・競合分析
2. **構成案作成**: `marketing/content-plan/` に企画書を作成
3. **下書き執筆**: `marketing/drafts/` で執筆（SEOガイドに準拠）
4. **レビュー・改善**: フィードバックを反映し清書
5. **公開**: `marketing/published/` にHTML出力
6. **リライト管理**: `marketing/revisions/` で更新履歴を管理

### SEO記事の品質基準

- 社内SEOガイド（`marketing/seo-guide.md`）に準拠
- 一次情報を優先（公式ドキュメント、GitHub等）
- 読者の「なぜ？」「どうやって？」を残さない
- フィードバックログ（`marketing/feedback-log.md`）に学びを蓄積

## リサーチの管理方法

リサーチは二層構造で管理しています。

| 層 | ディレクトリ | 内容 |
|----|-------------|------|
| 一次記録 | `research/topics/` | 情報源ごとの調査記録（1ソース1ファイル） |
| 統合ナレッジ | `research/knowledge/` | テーマ別に統合・整理したナレッジ |

**情報の優先順位**:
1. 一次情報（公式ドキュメント、GitHub）
2. 権威ある二次情報（Microsoft等の大手ベンダー記事）
3. 一般メディア・ブログ

## セットアップ

### 前提条件

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) がインストール済み
- [cc-company](https://github.com/Shin-sibainu/cc-company) プラグインがインストール済み

### 始め方

```bash
# 1. cc-company プラグインをインストール（初回のみ）
claude install-plugin https://github.com/Shin-sibainu/cc-company

# 2. リポジトリをクローン
git clone https://github.com/Bocek-Inc/openclaw-implementation-support.git
cd openclaw-implementation-support

# 3. Claude Codeを起動
claude

# 4. 秘書に話しかける
/company
```

### 新しいメンバーが参加する場合

1. リポジトリをクローン
2. Claude Codeを起動して `/company` を実行
3. 秘書が既存の組織構成を読み込んで応答します
4. 各部署の `CLAUDE.md` にルールが定義されているので、AIが自動的にルールに従います

## 運営ルール

| ルール | 内容 |
|--------|------|
| ファイル命名 | 日次: `YYYY-MM-DD.md`、トピック: `kebab-case.md` |
| テンプレート | 各フォルダの `_template.md` をコピーして使用 |
| 既存ファイル | 上書き禁止。追記のみ（タイムスタンプ付き） |
| 1トピック1ファイル | 情報が散らばらないよう徹底 |
| TODO形式 | `- [ ] タスク \| 優先度: 高/通常/低 \| 期限: YYYY-MM-DD` |
